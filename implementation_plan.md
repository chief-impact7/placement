# 트렌드 차트 UI 및 성능 최적화 계획 (v19)

## 1. 현재 문제 진단
- **데이터 로딩 부하**: 모든 학생 선택 시 수십 개의 차트가 동시에 데이터를 요청하여 지난 학기 성적을 불러오지 못하는 현상 발생.
- **UI 겹침 현상**: `sticky top-6` 설정으로 인해 스크롤 시 차트 카드들이 서로 겹쳐서 가려짐.

## 2. 작업 상세
### 2.1 App.jsx 수정
- **개수 제한**: `selectedStudents` 배열을 `slice(0, 4)`하여 최대 4명의 차트만 출력하도록 제한함.
- **레이아웃 수정**: 각 차트 컨테이너의 `sticky top-6` 클래스를 제거하여 일반적인 스크롤 흐름을 따르도록 함.

### 2.2 빌드 및 배포
- `npm run build` 및 `firebase deploy` 완료.

## 3. 검증 계획
- 학생을 5명 이상 선택해도 차트가 4개만 나오는지 확인.
- 스크롤 시 차트가 상단에 고정되어 겹치지 않고 정상적으로 올라가는지 확인.
